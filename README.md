## 核心原则

1. **显式优于隐式** — 永远显式指定，不要依赖默认值
2. **先搞清楚"不能做什么"** — 限制比能力更重要
3. **中间件是 bug 高发区** — 翻译必须完整
4. **精准比全面重要** — 少而准 > 多而乱
5. **自愈设计** — 假设进程会意外终止
6. **不要想当然** — 5 分钟读文档省 5 小时调试
7. **减法比加法安全** — 拿不准的时候，删掉它
8. **结构即美感** — 先把结构搞对
9. **终端是接口** — CLI 优先于 GUI
10. **精确的温度** — 尊重智商，不哄情绪
